## Environments
1. Node.js 默认是运行在开发环境的。设置成生产环境的话，主要有下面两个变化：
    * 最小化日志，只保留必要的内容
    * 更多的使用缓存以优化性能
2. 查询环境、改变环境以及其他细节，看 [这里](https://nodejs.dev/en/learn/nodejs-the-difference-between-development-and-production/)


## Misc
* 使用 `node` 命令运行一个 node 项目时，必须要在根目录运行。也就是说必须要在根目录 `node src/index`，而不能 src 目录 `node index`


## 升级为 ESM
### 添加后缀名
1. TypeScript 在 ESM 模式下要求显式扩展名。
2. 以前这样就可以
   ```ts
   import { helper } from './utils/helper
   ```
3. 现在必须改为
    ```ts
    import { helper } from './utils/helper.js'; // 注意是 .js 而不是 .ts
    ```
4. 可以使用下面的正则批量修改
    * 匹配：`(from\s+['"])(\..*?)(\.ts)?(['"])`
    * 替换为：`$1$2.js$4`
5. 注意如果路径如果只是 `.`（也就是引用同级的 `index.ts`），则会把路径替换为 `..js`。
