## 思想
1. 任何实体的设计，函数、模块或者是一个系统，都应该对它的使用者保持透明。
2. 也就是说，实体内部的改变，对于实体的使用者来说，应该是无感知的。
3. 黑箱满足使用者的意图，但是不需要使用者知道实现。


## 涉及的 bad code
* Data Class
* Insider Trading
* Long Parameter List
* Message Chains


## 涉及的重构方法
* Encapsulate Variable
* Encapsulate Collection
* Encapsulate Record
* Hide Delegate
* Replace Temp with Query
