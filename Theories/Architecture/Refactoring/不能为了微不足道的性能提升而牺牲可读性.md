# 不能为了微不足道的性能提升而牺牲可读性

## 美学
1. 与其说美学，倒不如说是经济学。也就是两种选择的成本与收益
2. 如果不是性能要求特别严格的场合，那为了很小的性能提升而牺牲更多的可读性是不值得的
3. 优雅很多时候比性能更重要！


## 例子
1. 三目运算符代码量会少一点点，而且可能也有一点点的性能优势。但如果像下面这样用，维护的成本就大于性能的收益了
    ```js
    if ( !aup || !bup ) {
        return a === doc ? -1 :
            b === doc ? 1 :
            aup ? -1 :
            bup ? 1 :
            sortInput ?
            ( indexOf.call( sortInput, a ) - indexOf.call( sortInput, b ) ) :
            0;
    }
    ```
    这是什么丑逼！！
